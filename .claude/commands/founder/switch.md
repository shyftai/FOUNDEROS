Switch to a different workspace.

Display:
```
<< FOUNDER:OS // SWITCH >>
```

Follow these steps:

1. Scan the workspaces/ directory for available workspaces
2. Display available workspaces:

```
  ┌─ WORKSPACES ────────────────────────────────────┐
  │                                                  │
  │  1. {workspace-name} — {stage}, {team size}      │
  │  2. {workspace-name} — {stage}, {team size}      │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

3. Ask which workspace to load, or accept the workspace name from the command argument
4. Load the workspace files (workspace.config.md, VISION.md, OKRS.md, PRIORITIES.md, DECISIONS.md, DELEGATION.md, ENERGY.md, BOARD.md, NETWORK.md, LEARNINGS.md, COSTS.md, ROADMAP.md, context/INDEX.md)
5. Re-detect connected OS frameworks
6. Display the loaded workspace confirmation:

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
┃  WORKSPACE: {Company Name}                                  ┃
┃  Stage: {stage}    Team: {size}    Role: {role}             ┃
┃  North star: {metric} = {current value}                     ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛
```
