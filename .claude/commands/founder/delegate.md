Assign work to people or OS frameworks. Route tasks to where they belong.

Display:
```
<< FOUNDER:OS // DELEGATE >>
```

Follow these steps:

1. Ask: "What task needs to be delegated?" — get a clear description
2. Classify the task by domain:
   - Pipeline, outbound, campaigns → GTM:OS
   - Content, publishing, brand → CONTENT:OS
   - Runway, invoicing, budget → FINANCE:OS
   - Hiring, team, culture → HR:OS
   - Fundraising, investors → INVESTOR:OS
   - Community, engagement → COMMUNITY:OS
   - Cross-domain / strategic → keep in FOUNDER:OS
3. Check the delegation matrix in DELEGATION.md:
   - Is this something only the founder should do?
   - Should it be delegated with oversight?
   - Can it be fully delegated?
4. Identify the assignee:
   - If routing to an OS framework: specify which framework and suggest the command to run
   - If routing to a person: specify who and what success looks like
5. Define the delegation:
   - Clear outcome expected
   - Due date
   - Check-in cadence (daily / weekly / on completion)
   - Decision authority (can they decide, or must they propose?)
6. Log the delegation in DELEGATION.md
7. Display:

```
  ┌─ DELEGATION ────────────────────────────────────┐
  │                                                  │
  │  Task:       {description}                       │
  │  Route to:   {person / OS framework}             │
  │  Due:        {date}                              │
  │  Authority:  {decide / propose}                  │
  │  Check-in:   {cadence}                           │
  │  OKR link:   {which key result}                  │
  │                                                  │
  │  Success looks like: {clear outcome}             │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

8. If routing to an OS framework, suggest the specific command:

```
  >> To execute: open {Framework} and run:
     /{command}:{sub-command} {workspace}
```

9. End with delegation status update showing total active delegations and any overdue items
