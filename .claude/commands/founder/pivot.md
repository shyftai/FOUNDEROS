Structured pivot analysis. Use when questioning whether to change direction.

Display:
```
<< FOUNDER:OS // PIVOT ANALYSIS >>
```

Follow these steps:

1. Ask: "What are you considering pivoting? Be specific about what would change."
2. Read VISION.md, OKRS.md, LEARNINGS.md, DECISIONS.md
3. Classify the pivot:
   - **Product pivot** — same market, different solution
   - **Market pivot** — same product, different market
   - **Business model pivot** — same product and market, different monetization
   - **Channel pivot** — same product, different distribution
   - **Full pivot** — new market and new solution (most risky)
4. Pull data to assess the current state:
   - GTM:OS → current traction metrics, what's working vs not
   - FINANCE:OS → runway remaining (how long can you explore?)
   - HR:OS → team capabilities (can the current team execute the pivot?)
   - COMMUNITY:OS → customer feedback and sentiment
5. Run the pivot framework:

   **Evidence for pivoting:**
   - What data shows the current direction isn't working?
   - How long have you tried? (minimum 6 months for B2B before pivoting)
   - Have you exhausted iterations on the current approach?

   **Evidence against pivoting:**
   - What's working that you'd lose?
   - What early signals of traction exist?
   - Are you pivoting because it's hard or because the data says so?

   **Pivot cost analysis:**
   - Runway impact: how many months does the pivot cost?
   - Team impact: can the current team execute? Do you need new hires?
   - Customer impact: what happens to existing users/customers?
   - Investor impact: how will investors react?

6. This is a **Type 1 decision** — run all six quality gates
7. Display the analysis:

```
  ┌─ PIVOT ANALYSIS ────────────────────────────────────────┐
  │                                                          │
  │  Type: {product/market/model/channel/full}               │
  │                                                          │
  │  Evidence FOR pivot:                                     │
  │  - {evidence 1}                                          │
  │  - {evidence 2}                                          │
  │                                                          │
  │  Evidence AGAINST pivot:                                 │
  │  - {evidence 1}                                          │
  │  - {evidence 2}                                          │
  │                                                          │
  │  Pivot cost:                                             │
  │    Runway: {months burned}  Team: {impact}               │
  │    Customers: {impact}  Investors: {impact}              │
  │                                                          │
  │  ── DECISION GATES ────────────────────────────────      │
  │  [x] Vision       [x] OKR impact    [x] Data backed     │
  │  [ ] Reversible   [x] Delegation    [x] Energy           │
  │  ─────────────────────────────────────────────────       │
  │                                                          │
  │  Recommendation: {pivot / iterate / wait}                │
  │                                                          │
  └──────────────────────────────────────────────────────────┘
  >> Pivot / Iterate instead / Defer decision
```

8. Log the decision in DECISIONS.md regardless of outcome
9. If pivoting: update VISION.md, OKRS.md, PRIORITIES.md, and ROADMAP.md
10. Log learnings about what didn't work in LEARNINGS.md (anti-learnings)
