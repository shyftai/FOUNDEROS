Product or campaign launch checklist. Coordinates across all relevant OS frameworks.

Display:
```
<< FOUNDER:OS // LAUNCH >>
```

Follow these steps:

1. Ask: "What are you launching?" (product, feature, campaign, partnership, announcement)
2. Ask: "When is the target launch date?"
3. Build the launch checklist based on type, pulling from connected frameworks:

**Pre-launch (2-4 weeks before):**
- [ ] Launch goal defined and tied to OKR
- [ ] Success metrics defined (what does good look like?)
- [ ] CONTENT:OS → content calendar updated, launch content created
- [ ] GTM:OS → outbound campaign prepped, list built, copy written
- [ ] COMMUNITY:OS → community primed, teaser posted
- [ ] HR:OS → team briefed, roles assigned for launch day
- [ ] FINANCE:OS → budget allocated, spend approved

**Launch readiness (1 week before):**
- [ ] All content approved and scheduled
- [ ] All campaigns loaded and ready to activate
- [ ] Community announcements drafted
- [ ] Internal team briefed with talking points
- [ ] Investor update drafted (if significant milestone)
- [ ] Press/announcement ready (if public)
- [ ] Rollback plan defined (if applicable)

**Launch day:**
- [ ] All systems go — activate campaigns
- [ ] Community announcement posted
- [ ] Content published
- [ ] Team monitoring channels
- [ ] Founder available for escalations

**Post-launch (1-2 weeks after):**
- [ ] Metrics collected from all OS frameworks
- [ ] Win/miss analysis
- [ ] Learnings logged to LEARNINGS.md
- [ ] Follow-up actions identified

4. Display the launch dashboard:

```
  ┌─ LAUNCH CHECKLIST ──────────────────────────────────────┐
  │                                                          │
  │  Launch:  {what}                                         │
  │  Date:    {when}                                         │
  │  OKR:     {which key result}                             │
  │                                                          │
  │  Pre-launch:    {n}/{total} complete                     │
  │  Readiness:     {n}/{total} complete                     │
  │  Launch day:    {n}/{total} complete                     │
  │  Post-launch:   {n}/{total} complete                     │
  │                                                          │
  │  Status:  {ready / not ready / blocked}                  │
  │  Blocked: {what's blocking, if anything}                 │
  │                                                          │
  └──────────────────────────────────────────────────────────┘
```

5. Track launch items in DELEGATION.md with owners and due dates
6. If public announcement: flag as HARD GATE requiring approval
7. After launch: prompt for /founder:debrief to capture learnings
