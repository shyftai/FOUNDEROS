# Board Guide

Reference for board meeting preparation, update formats, and governance. Load before running `/founder:board-update`.

---

## Board update format

Every board update follows this structure. Keep it concise — board members are busy. 2-3 pages maximum.

### 1. TL;DR (3 sentences max)
The entire update in a nutshell. A board member who reads nothing else should get the picture.

### 2. Highlights (3-5 bullets)
What went well. Milestones hit. Wins to celebrate.

### 3. Key Metrics Table

| Metric | Last Period | This Period | Target | Status |
|--------|-----------|-------------|--------|--------|
| Revenue (MRR/ARR) | | | | |
| Pipeline (weighted) | | | | |
| Burn rate (monthly) | | | | |
| Runway (months) | | | | |
| Team size | | | | |
| North star metric | | | | |
| Customer count | | | | |
| Churn rate | | | | |

Status: on-track / at-risk / behind

### 4. OKR Progress
One line per key result with score.

### 5. Challenges (2-3 bullets)
What's not working. Be honest — boards respect candor over spin. For each challenge, state what you're doing about it.

### 6. Asks (1-2 specific requests)
- Be specific: "Intro to Jane Smith, CTO at TechCorp" not "help with partnerships"
- Make it easy to help: provide context for why this intro matters

### 7. Next Period Outlook (3-5 bullets)
Top priorities, key hires, expected milestones.

## Board meeting preparation checklist

**2 weeks before:**
- [ ] Confirm date, time, agenda with board members
- [ ] Begin compiling data from all OS frameworks
- [ ] Draft board update

**1 week before:**
- [ ] Finalize board update
- [ ] Prepare any supplementary materials (financial model, product demo, hiring plan)
- [ ] Send materials to board (default: 5 days before meeting)
- [ ] Prepare talking points for challenges section (anticipate questions)

**Day before:**
- [ ] Review all materials one final time
- [ ] Prepare answers for likely tough questions
- [ ] Test any technology (video call, screen share, slides)

**Meeting day:**
- [ ] Arrive early / log in early
- [ ] Have data from all OS frameworks accessible for live questions
- [ ] Take notes on action items and commitments

**After meeting:**
- [ ] Send meeting summary within 24 hours
- [ ] Log all action items in DELEGATION.md
- [ ] Log decisions in DECISIONS.md
- [ ] Update BOARD.md communication log

## What boards want to see

### Early stage (pre-seed / seed)
- Product progress and user feedback
- Key learnings and pivots
- Runway and next fundraise timing
- Founder well-being (boards care about this)
- Specific help needed (intros, advice)

### Growth stage (series A+)
- Revenue growth and unit economics
- Pipeline and sales efficiency
- Hiring progress and organizational health
- Competitive landscape
- Strategic decisions and rationale
- Cash management and fundraise planning

## Tone and style

- **Be concise** — board members read dozens of updates. Respect their time.
- **Lead with data** — numbers before narrative
- **Be honest** — bad news doesn't get better with time. Surface issues early.
- **Be specific in asks** — "Can you introduce me to X?" not "Help with sales"
- **Show learning** — demonstrate that you're adapting based on results
- **Never surprise** — if something material changes, communicate between updates

## Board governance basics

### Typical board consent items
These usually require board approval (check your shareholder agreement):
- Fundraising and issuing new shares
- Annual budget approval
- Executive compensation
- Major contracts (>$X threshold)
- Pivots or major strategic changes
- M&A activity
- Debt financing

### Information rights
Most investors with board seats are entitled to:
- Monthly or quarterly financial statements
- Annual budget and forecast
- Cap table access
- Material event notifications

### Board communication cadence
| Type | Cadence | Format |
|------|---------|--------|
| Board update | Quarterly (or per meeting cadence) | Written report |
| Board meeting | Quarterly | Video/in-person |
| Ad-hoc update | As needed | Email |
| Annual strategy | Yearly | Presentation + discussion |

## Common board meeting pitfalls

| Pitfall | Why it's bad | Fix |
|---------|-------------|-----|
| Too much detail | Loses the forest for the trees | Lead with summary, offer detail in appendix |
| No challenges section | Looks like you're hiding problems | Always include 2-3 honest challenges |
| Vague asks | Board can't help if they don't know how | Be specific: name, company, context |
| Reading slides | Boring and wasteful | Send materials ahead, discuss only |
| No action items | Meeting has no output | End every meeting with clear next steps |
| Defensive on challenges | Breaks trust | Accept feedback, show learning |
